Identification of brain region responsible for responding to a stimulus using GLM on fMRI data

Humans responding to stimulus is a well known fact. However, detecting which region in the brain is responsible for producing the response is hard task. At any given time the brain will be always be functioning to carry out the basic metabolic activities such as breathing, digestion, controlling the heart-rate and body temperature. fMRI scans provides scientists with valuable information of which region in the brain is active by indirectly measuring the magnetic properties of oxygen in fresh oxygenated blood and deoxygenated blood. Using GLM, attempts are made to remove the noise of background active neurons and detect the regions of the brain. Standard fMRI data sets were used.

A 1D fast fourier tranform on CPU vs GPU: Who will win?

It is known that GPUs are massive parallel computing machines that can perform matrix operations on a large data very efficiently. CUDA from Nvidia provides libraries for performing FFT on the GPU and OpenCL also provides the FFT libraries. With the GPU disabled and it's drivers removed from the system, the FFT is performed on the CPU. However, the results show that the FFT applied on an image matrix transformed into 1D matrix is faster on a CPU than on the GPU. The results suggest that Linux needs to improve it's DMA support for the GPU because the data transfer between the CPU and GPU is found to be responsible for this poor performance of GPUs.

A secure Online Code Judge without sandbox

A sandbox for running code received as input seems to be a critical component in many online code judges. However, it is possible to bypass the need for sandboxes in compiled languages such as C/C++ and some interpreted languages such as Python by copying the library files into a chroot jail. The chroot jail has two components. The first is that it only contains the basic libraries that are needed for a program to run. The second innovative component is to edit the header files for functions that are not needed by the program. This breaks the connection that the interface provides between the runtime and the program. The program cannot find the function definition in the interface and thus, is not able to call the function present in the runtime library. This method is very efficient when compared to emulating a sandbox.
